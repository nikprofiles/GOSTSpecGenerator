# GOSTSpecGenerator — Контекст проекта

## Суть проекта

Десктопное приложение (Windows .exe) для автоматической генерации PDF-спецификаций оборудования, изделий и материалов по ГОСТ 21.110-2013. Берёт данные из Excel, раскладывает на листы с рамками и штампами по ГОСТ 21.101, выводит готовый PDF на печать.

**Целевая аудитория:** проектировщики, инженеры ПГС/ЭС/ОВ, студенты технических вузов.

**Репозиторий:** https://github.com/nikprofiles/GOSTSpecGenerator
**Текущая версия:** v1.0.9 (17.03.2026)

## Известные проблемы и ограничения

### Критические
1. **Нет undo/redo** — после вставки/удаления строки нельзя отменить действие
2. **Нет сохранения проекта** — при закрытии приложения все настройки штампа теряются (нужно заново вводить)

### Средние
3. **Шрифт ISOCPEUR** — не бандлен (лицензия Autodesk). Используется Arial как fallback. ГОСТ требует шрифт типа Б — нужно найти свободный аналог
4. **Bold/Italic в PDF** — имитация bold через двойную отрисовку со сдвигом 0.3pt (нет отдельного bold-варианта Arial). Italic использует тот же шрифт, если ISOCPEUR_Italic.ttf отсутствует
5. **Эмблема** — масштабирование работает, но нет обрезки/кадрирования. Изображение вписывается "как есть"
6. **Автоопределение типов строк** — эвристика (1-2 заполненных текстовых поля = категория). Может ошибаться на нестандартных таблицах

### Мелкие
7. **Кодировка путей** — bash на Windows с кириллическими путями выдаёт "Permission denied" (не влияет на работу, артефакт оболочки)
8. **Размер .exe** — 80МБ (PySide6 + ReportLab + PyMuPDF). Можно уменьшить через --onedir + UPX

## Идея: интеграция Claude API (LLM-модуль)

### Концепция
Подключить Claude API через ключ для интеллектуальной обработки входных данных. Пользователь загружает "сырой" Excel без строгой структуры — LLM анализирует содержимое и формирует структурированную спецификацию.

### Что может LLM:
1. **Классификация строк** — определить, что "Этаж -1" это категория, "Низковольтное оборудование" — подкатегория, остальное — данные. Не по формальным правилам, а по смыслу.
2. **Маппинг колонок** — если в Excel колонки называются нестандартно ("Название", "Артикул", "Цена за шт."), LLM сопоставит их с 9 колонками ГОСТ.
3. **Генерация из ТЗ** — пользователь вводит текст "нужна спецификация на электрику для 3-этажного жилого дома", LLM генерирует шаблон.
4. **Автозаполнение** — по наименованию оборудования подтянуть тип/марку, код продукции, производителя из базы знаний.
5. **Валидация** — проверка на дубликаты, нелогичные значения, пропущенные позиции.

### Реализация:
- Кнопка "Умная обработка" в тулбаре (опциональная, требует API-ключ)
- Настройки → поле "Claude API Key"
- Модуль `core/llm_processor.py` — отправляет данные в Claude, получает структурированный JSON
- Fallback: если нет ключа или нет интернета — работает по правилам (как сейчас)

### Стоимость:
- Claude API: ~$3/1M input tokens, ~$15/1M output tokens
- Средняя спецификация: ~2000 tokens → ~$0.01-0.03 за обработку
- Можно добавить счётчик "осталось запросов" для контроля бюджета

## Контекст для восстановления сессии

- PySide6 вместо PyQt6 (Python 3.14 не поддерживает PyQt6)

- Штамп по скриншотам заказчика (не строго по ГОСТ — адаптация)
- Формат под рамкой, не внутри штампа
- Автонумерация по правилам, без AI (пока)
