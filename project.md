# GOSTSpecGenerator — Контекст проекта

## Суть проекта

Десктопное приложение (Windows .exe) для автоматической генерации PDF-спецификаций оборудования, изделий и материалов по ГОСТ 21.110-2013. Берёт данные из Excel, раскладывает на листы с рамками и штампами по ГОСТ 21.101, выводит готовый PDF на печать.

**Целевая аудитория:** проектировщики, инженеры ПГС/ЭС/ОВ, студенты технических вузов.

**Репозиторий:** https://github.com/nikprofiles/GOSTSpecGenerator
**Текущая версия:** v1.1.7 (17.03.2026)

---

## Технологический стек

| Компонент | Технология |
|-----------|-----------|
| Язык | Python 3.10+ |
| PDF-генерация | ReportLab (Canvas API, точность до 0.35мм) |
| Чтение Excel | openpyxl |
| GUI | PySide6 (Qt6) |
| Предпросмотр PDF | PyMuPDF (fitz) |
| Сборка .exe | PyInstaller (--onefile, ~80МБ) |
| Шрифты | Arial (бандлен), ISOCPEUR, Times New Roman (системные) |

## Известные проблемы и ограничения

### Средние
1. **Bold/Italic в PDF** — имитация bold через двойную отрисовку (нет отдельного bold-TTF)
2. **Эмблема** — масштабирование без обрезки/кадрирования
3. **Автоопределение типов строк** — эвристика, может ошибаться на нестандартных таблицах

### Мелкие
4. **Кодировка путей** — bash на Windows с кириллическими путями выдаёт "Permission denied" (артефакт)
5. **Размер .exe** — 80МБ (PySide6 + ReportLab + PyMuPDF)

## Контекст для восстановления сессии

**Заказчик:** Никита Егоров (@nikprofiles), проектировщик. Работает со спецификациями по ГОСТ для строительных проектов (ЭС, ОВ, ВК).

**Ключевые решения:**
- PySide6 вместо PyQt6 (Python 3.14 не поддерживает PyQt6)
- 9 колонок (не 10): "Кол." одна колонка, "Поставщик" вместо "Завод-изготовитель"
- Роли настраиваемые (не фиксированные)
- Штамп по скриншотам заказчика (адаптация)
- 3-уровневая нумерация: Категория → Подкатегория → Данные

**Рабочая директория:** `C:\GOSTSpec\`
**Сборка:** `pyinstaller gost_spec.spec --clean --noconfirm`
